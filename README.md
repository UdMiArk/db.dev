База корпоративных ресурсов (изображений, инструкций, шаблонов для печати и т.п.)

Разрабатывалась для работы в жесткой связке с корпоративным сервером AD и местной ERP
 для аутентификации и определения доступных для пользователя проектов. Без них бесполезна,
 так как для регистрации ресурса в базе необходимо указать связанный проект (объект продвижения).
Выложена как вариант демонстрации кода.

Без настроенного LDAP соединения с подходящим сервером и доступного ERP можно залогиниться
 только тестовым пользователем и только при включенном режиме отладки (YII_DEBUG == true).
Логин тестового пользователя - 'test.user', домен и пароль - любые.
Для доступа к более интересным функциям интерфейса ему стоит выдать роль администратора через CLI:\
`yii service/assign-admin test.user`
